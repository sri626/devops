What is SDLC?
SDLC (Software Development Life Cycle) is a step-by-step process used to design, develop, test, and deploy software.
Think of it like building a house:
•	You don’t start with construction immediately 
•	First, you plan, design, build, inspect, and then hand it over 
Software development works in the same way.

Why is SDLC Important?
Without SDLC, software development becomes chaotic.
SDLC helps in:
•	Avoiding confusion and mistakes 
•	Managing time and cost effectively 
•	Improving software quality 
•	Ensuring customer satisfaction 
•	Making teamwork easier 
In short, SDLC brings discipline and structure to development.

Phases of SDLC
SDLC consists of several stages. Each stage has a specific purpose.

1. Requirement Gathering & Analysis
This is the first and most important phase.
Here, the team:
•	Talks to clients or stakeholders 
•	Understands what the software should do 
•	Collects all requirements 
Example:
If building a food delivery app, requirements may include:
•	User login 
•	Restaurant listing 
•	Payment system 
 If this step is done poorly, the entire project can fail.

2. Planning
In this phase, the team creates a roadmap.
They decide:
•	Project timeline 
•	Budget 
•	Resources needed 
•	Risks involved 
This is like answering:
“How are we going to build this?”

3. Design
Now the system structure is created.
This includes:
•	Architecture design 
•	Database design 
•	UI/UX design 
Two types of design:
•	High-Level Design (HLD) – overall system structure 
•	Low-Level Design (LLD) – detailed component design 
Example:
Designing how users interact with an app screen.

4. Development (Coding)
This is where developers start writing code.
•	Each module is built 
•	Features are implemented 
•	Developers follow coding standards 
This is the longest phase of SDLC.

5. Testing
Once the software is built, it must be tested.
Testing ensures:
•	The software works correctly 
•	Bugs are identified and fixed 
•	Performance is stable 
Types of testing include:
•	Unit Testing 
•	Integration Testing 
•	System Testing 
•	User Acceptance Testing (UAT) 
👉 Good testing = better quality software.

6. Deployment
After testing, the software is released to users.
Deployment can be:
•	Full release (all at once) 
•	Gradual release (phases) 
Example:
Launching an app on Play Store or App Store.

7. Maintenance
Even after release, the work is not over.
Maintenance includes:
•	Fixing bugs 
•	Adding new features 
•	Improving performance 
This phase ensures the software stays useful over time.

Different SDLC Models
There are different ways to follow SDLC depending on the project.

1. Waterfall Model
•	Linear and step-by-step 
•	Each phase is completed before moving to the next 
•	Best for simple, fixed projects 

2. Agile Model
•	Flexible and iterative 
•	Work is divided into small cycles (sprints) 
•	Continuous feedback and improvement 
👉 Most modern companies use Agile.

3. Spiral Model
•	Combines design and risk analysis 
•	Focuses on minimizing risks 
•	Used in complex projects 

4. V-Model
•	Testing is done alongside development 
•	Each development phase has a corresponding testing phase 

Real-Life Example of SDLC
Let’s say you are building an online shopping app:
1.	Requirement – Users should browse, add to cart, and pay 
2.	Planning – Decide timeline and team 
3.	Design – Create app layout and database 
4.	Development – Write code for features 
5.	Testing – Check login, payments, and bugs 
6.	Deployment – Launch app 
7.	Maintenance – Fix issues and add new features
